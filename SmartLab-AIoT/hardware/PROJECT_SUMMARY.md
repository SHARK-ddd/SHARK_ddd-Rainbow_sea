## 二、硬件配置

### 2.1 传感器接线

| 传感器引脚 | ESP32-S3 引脚 | 功能 |
|------------|---------------|------|
| VCC | 3.3V | 电源输入 |
| GND | GND | 接地 |
| AO（模拟输出） | GPIO3 (ADC1_CH3) | 光照模拟信号 |
| DO（数字输出） | 未使用 | 预留接口 |

### 2.2 引脚选择说明

**关键经验**：
- ESP32-S3 的 GPIO34-GPIO39 属于 RTC GPIO，在部分 MicroPython 固件中可能受限
- **推荐使用 ADC1 通道**（GPIO1-GPIO11），兼容性更好
- 本项目最终选择 **GPIO3**，经测试稳定可靠

## 四、问题排查与解决

### 4.1 问题清单

| 序号 | 问题描述 | 错误代码 | 根因 | 解决方案 |
|------|----------|----------|------|----------|
| 1 | 引脚初始化失败 | `ValueError: invalid pin` | RTC GPIO 受限 | 改用 GPIO3 |
| 2 | MQTT 连接失败 | `ECONNABORTED (113)` | 防火墙阻止 | 添加入站规则 |
| 3 | 文件缺失 | `ImportError: no module named 'sensor'` | 未上传 sensor.py | 上传所有文件 |
| 4 | 端口未监听 | 连接超时 | Mosquitto 配置问题 | 添加 listener 配置 |

### 4.2 排查流程

```
MQTT连接失败
    ↓
┌─────────────────────────────────────┐
│ 1. 确认 WiFi 连接成功              │
│    → 检查 IP 地址是否正确           │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 确认端口监听状态                │
│    → netstat -ano | findstr :1883  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 确认防火墙规则                  │
│    → 检查入站规则是否允许 1883     │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 确认 Mosquitto 配置            │
│    → listener 1883                │
└─────────────────────────────────────┘
```

### 4.3 关键配置

**Mosquitto 配置文件（mosquitto.conf）**：
```ini
listener 1883          # 监听所有接口的 1883 端口
allow_anonymous true   # 允许匿名连接（开发环境）
```

**Windows 防火墙规则**：
```powershell
New-NetFirewallRule -Name "Mosquitto_MQTT" `
    -Direction Inbound -Protocol TCP `
    -LocalPort 1883 -Action Allow
```

## 五、学习要点与经验总结

### 5.1 ESP32 开发要点

| 要点 | 说明 |
|------|------|
| **引脚兼容性** | 优先使用 ADC1 通道（GPIO1-11），避免 RTC GPIO |
| **固件差异** | 不同 MicroPython 固件对 GPIO 的支持可能不同 |
| **文件上传** | 确保所有依赖文件完整上传到设备 |
| **调试技巧** | 使用 Thonny 串口终端查看实时日志 |

### 5.2 MQTT 协议知识

| 概念 | 说明 |
|------|------|
| **Broker** | 消息代理服务器，负责转发消息 |
| **Topic** | 消息主题，支持通配符订阅（`#` 和 `+`） |
| **QoS** | 服务质量等级（0/1/2） |
| **Keepalive** | 心跳间隔，维持连接状态 |

### 5.3 网络排查经验

1. **确认网络连通性**：ESP32 和服务器必须在同一网段
2. **检查端口监听**：使用 `netstat` 确认端口是否开放
3. **防火墙配置**：入站规则必须允许目标端口
4. **服务状态**：确认 Mosquitto 服务正在运行

### 5.4 开发工具链

| 工具 | 用途 |
|------|------|
| Thonny | MicroPython 开发环境，代码上传与调试 |
| Mosquitto | MQTT Broker 服务器 |
| PowerShell | Windows 命令行，服务管理 |
| Python | 编写诊断工具和订阅器 |

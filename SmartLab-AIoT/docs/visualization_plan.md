# SmartLab-AIoT 数据可视化方案规划

## 一、当前数据格式分析

### 1.1 MQTT 订阅消息格式
当前接收到的 MQTT 消息格式为：
```json
{
  "device_id": "esp32_s3_e072", 
  "timestamp": 833407800, 
  "light": 80.98, 
  "humidity": 38, 
  "temperature": 25
}
```

### 1.2 数据字段说明
- `device_id`: 设备唯一标识符（如 esp32_s3_e072）
- `timestamp`: 时间戳（Unix 时间戳格式）
- `light`: 光照强度（单位：lux）
- `humidity`: 湿度（单位：%）
- `temperature`: 温度（单位：°C）

### 1.3 数据存储现状
- **SQLite 数据库**: 已实现基础存储
- **InfluxDB**: 已集成但未完全启用（Windows 文件锁问题）
- **后端 API**: 已实现数据查询接口

## 二、可视化需求分析

### 2.1 核心需求
1. **实时数据监控**: 显示当前传感器数据
2. **历史数据趋势**: 展示时间序列数据变化
3. **设备状态管理**: 监控多个设备的状态
4. **异常报警**: 数据异常时发出警告
5. **数据导出**: 支持数据导出功能

### 2.2 用户角色
- **实验室管理员**: 需要全面的数据监控和报警功能
- **研究人员**: 需要历史数据分析和趋势查看
- **维护人员**: 需要设备状态监控和故障诊断

## 三、技术方案设计

### 3.1 前端技术栈
- **框架**: Vue 3 + Element Plus
- **状态管理**: Pinia
- **图表库**: ECharts / Chart.js
- **实时通信**: WebSocket
- **HTTP 客户端**: Axios

### 3.2 后端接口规划
#### 3.2.1 实时数据接口
- `GET /api/realtime/data`: 获取最新传感器数据
- `GET /api/realtime/status`: 获取设备连接状态

#### 3.2.2 历史数据接口
- `GET /api/history/data?device_id=xxx&start_time=xxx&end_time=xxx`: 获取历史数据
- `GET /api/history/aggregated?device_id=xxx&period=hourly`: 获取聚合数据

#### 3.2.3 设备管理接口
- `GET /api/devices`: 获取设备列表
- `GET /api/devices/{device_id}`: 获取设备详情
- `POST /api/devices`: 添加新设备

### 3.3 实时通信方案
- **WebSocket**: 实现服务器推送实时数据
- **心跳机制**: 维持连接稳定性
- **数据缓存**: 前端本地缓存最近数据

|----------------------------------------|

### 4.2 核心组件设计

#### 4.2.1 实时数据卡片
- 显示当前温度、湿度、光照数值
- 使用不同颜色表示正常/警告/危险状态
- 支持设备切换

#### 4.2.2 趋势图表
- 折线图：显示24小时数据趋势
- 柱状图：显示各设备数据对比
- 面积图：显示数据变化范围

#### 4.2.3 设备管理
- 设备列表：显示所有连接设备
- 设备详情：查看单个设备历史数据
- 设备控制：支持远程配置（未来扩展）

### 4.3 响应式设计
- **桌面端**: 完整仪表盘布局
- **平板端**: 简化布局，重点数据突出
- **移动端**: 卡片式布局，关键信息优先

## 六、技术难点与解决方案

### 6.1 实时数据更新
- **问题**: WebSocket 连接稳定性
- **解决方案**: 
  - 实现重连机制
  - 本地数据缓存
  - 心跳检测

### 6.2 大数据量处理
- **问题**: 历史数据查询性能
- **解决方案**:
  - 分页加载
  - 数据聚合
  - 懒加载策略

### 6.3 跨设备兼容
- **问题**: 不同屏幕尺寸适配
- **解决方案**:
  - 响应式布局
  - 移动端优化
  - 触摸友好设计
